## Architecture Overview

GEOL (Geographic Explorer & Oracle of Locations) is a single-page React app that pairs an interactive Google Map with an AI chat interface. It was built for the NYC CBS / Google Hackathon.

### Frontend (`src/`)

- **`src/app/App.tsx`** â€” The entire application state lives here. It manages location flow (idle â†’ asking â†’ detecting/manual â†’ ready), chat message history, and coordinates all child components. All API calls are made from this file except for map-specific ones in `LocationMap.tsx`.
- **`src/app/components/LocationMap.tsx`** â€” Google Maps rendered via `@vis.gl/react-google-maps`. Displays a 5-mile radius circle, user position marker, and POI markers. Fetches nearby places from the backend when position changes.
- **`src/app/components/ChatPanel.tsx`** â€” Chat message list with support for structured `info` payloads (facts, events, real estate listings, historical image), suggested follow-up questions, and location onboarding prompts.
- **`src/app/components/VoiceAssistant.tsx`** â€” Browser Web Speech API for voice input; speaks assistant responses aloud.
- **`src/app/components/ui/`** â€” shadcn/ui components (Radix UI primitives + Tailwind). Do not modify these directly.

### Backend (`supabase/functions/server/`)

A Deno-based [Hono](https://hono.dev/) server deployed as a Supabase Edge Function. All routes are prefixed with `/make-server-3c4885b3`.

| Endpoint | Description |
|---|---|
| `GET /maps-key` | Returns `GOOGLE_MAPS_API_KEY` to the client |
| `POST /geocode` | Resolves addresses/coordinates using Gemini AI (no Geocoding API required) |
| `POST /nearby-places` | Fetches POIs via Google Places API (New) |
| `POST /chat` | Main conversational AI endpoint; sends full message history to Gemini |
| `POST /place-info` | AI-generated facts about a clicked map marker |
| `POST /generate-image` | Generates historical images via Imagen 3 or Gemini Flash |
| `POST /historical-streetview` | Fetches current Street View + generates 1920s-era AI rendering |

**Gemini model fallback chain** (defined in `GEMINI_TEXT_MODELS`): `gemini-2.0-flash` â†’ `gemini-2.0-flash-lite` â†’ `gemini-1.5-flash-8b`. Rate-limited (429) models are skipped automatically. Image generation tries `imagen-3.0-generate-002` â†’ `imagen-3.0-generate-001` â†’ `gemini-2.0-flash-exp`.

**Required environment variables** (set in Supabase dashboard, not in `.env`):
- `GEMINI_API_KEY`
- `GOOGLE_MAPS_API_KEY` (must have Maps JavaScript API, Places API New, Street View Static API, and Geocoding API enabled)

### Chat message shape

- `info` â€” structured data (facts, historicalFacts, events, realEstate, historicalImagePrompt)
- `suggestedQuestions` â€” follow-up prompts rendered as clickable chips
- `locationPrompt` â€” renders location detection buttons instead of text

- `generatedImage` / `imageLoading` â€” AI-generated historical image state
